# Quickstart

Build a minimal Nitrolite script that connects to Nitronode, signs channel states with your wallet, deposits USDC into a home channel, sends an off-chain transfer, and closes the connection cleanly.

:::info Sandbox URL - coming soon
The canonical public Nitronode sandbox WebSocket URL is still being pinned by the ops team. Use the literal placeholder shown below, then replace it in the WebSocket URL line only when the final hostname is published.
:::

:::tip Migrating from 0.5.3?
Already on `@erc7824/nitrolite@0.5.3`? Skip ahead to [Migrating from 0.5.3](../sdk/typescript-compat/overview) - you'll probably want `@yellow-org/sdk-compat` plus the `@yellow-org/nitrolite-codemod`, not this guide.
:::

## Prerequisites

Before you start, make sure you have:

- Node.js 20 or later.
- A development wallet private key with testnet gas.
- A blockchain RPC endpoint for Polygon Amoy or your target chain.
- `decimal.js` for exact asset amounts.
- `viem` for wallet and address types used by the SDK.

See [Prerequisites & Environment](./prerequisites) for a full setup checklist.

## Step 1: Install

Create a new TypeScript project and install the v1 SDK:

```bash
mkdir yellow-quickstart
cd yellow-quickstart
npm init -y
npm install @yellow-org/sdk decimal.js viem dotenv
npm install -D typescript @types/node
```

Set the project to ESM and add a build script:

```json title="package.json"
{
  "type": "module",
  "scripts": {
    "build": "tsc"
  }
}
```

Add a minimal `tsconfig.json`:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "outDir": "dist"
  },
  "include": ["src/**/*"]
}
```

## Step 2: Create signers

`createSigners()` builds the two signer objects the client needs: one for off-chain channel states and one for blockchain transactions.

```typescript title="src/index.ts"
import 'dotenv/config';
import { Client, createSigners, withBlockchainRPC } from '@yellow-org/sdk';
import Decimal from 'decimal.js';

const PRIVATE_KEY = process.env.PRIVATE_KEY as `0x${string}`;
const RECIPIENT = process.env.RECIPIENT as `0x${string}`;
const RPC_URL = process.env.RPC_URL;
const CHAIN_ID = 80002n; // Polygon Amoy
const ASSET = 'usdc';

if (!PRIVATE_KEY) throw new Error('Set PRIVATE_KEY in .env');
if (!RECIPIENT) throw new Error('Set RECIPIENT in .env');
if (!RPC_URL) throw new Error('Set RPC_URL in .env');

const { stateSigner, txSigner } = createSigners(PRIVATE_KEY);

console.log('Wallet address:', stateSigner.getAddress());
```

## Step 3: Connect

Create the client with a Nitronode WebSocket URL, both signers, and a chain RPC endpoint for on-chain settlement.

```typescript title="src/index.ts"
const NITRONODE_WS_URL = '<sandbox-url-coming-soon>'; // Replace with the canonical sandbox URL.

const client = await Client.create(
  NITRONODE_WS_URL,
  stateSigner,
  txSigner,
  withBlockchainRPC(CHAIN_ID, RPC_URL),
);

await client.setHomeBlockchain(ASSET, CHAIN_ID);
console.log('Connected as:', client.getUserAddress());
```

## Step 5: Deposit

Deposit builds and co-signs the next channel state off-chain. `checkpoint()` submits that state to the ChannelHub contract so the deposit is enforced on-chain.

```typescript title="src/index.ts"
await client.approveToken(CHAIN_ID, ASSET, new Decimal(5));

const depositState = await client.deposit(CHAIN_ID, ASSET, new Decimal(5));
console.log('Deposit state version:', depositState.version);

const depositTx = await client.checkpoint(ASSET);
console.log('Deposit checkpoint tx:', depositTx);
```

`approveToken()` is only needed before the first checkpoint for a token, or when you need to increase allowance.

## Step 6: Transfer

Transfers advance the signed off-chain state. They do not need a checkpoint unless you want to settle the latest state on-chain.

```typescript title="src/index.ts"
const transferState = await client.transfer(
  RECIPIENT,
  ASSET,
  new Decimal(1),
);

console.log('Transfer transition:', transferState.transition.type);
```

## Step 7: Clean up

Always close the client when your script is done:

```typescript title="src/index.ts"
await client.close();
```

A complete script should wrap the channel operations in `try` / `finally` so the WebSocket closes even if a request fails.

```typescript title="src/index.ts"
try {
  await client.approveToken(CHAIN_ID, ASSET, new Decimal(5));

  const depositState = await client.deposit(CHAIN_ID, ASSET, new Decimal(5));
  console.log('Deposit state version:', depositState.version);

  const depositTx = await client.checkpoint(ASSET);
  console.log('Deposit checkpoint tx:', depositTx);

  const transferState = await client.transfer(
    RECIPIENT,
    ASSET,
    new Decimal(1),
  );
  console.log('Transfer transition:', transferState.transition.type);
} finally {
  await client.close();
}
```

Run it after the sandbox URL is available and your `.env` values are set:

```bash
npm run build
node dist/index.js
```

## What just happened

- `createSigners()` derived wallet-backed signers for off-chain states and on-chain transactions.
- `Client.create()` opened a v1 Nitronode WebSocket connection and attached a blockchain RPC endpoint for `checkpoint()`.
- `deposit()` prepared a mutually signed channel state, then `checkpoint()` enforced it on-chain through ChannelHub.
- `transfer()` advanced the off-chain channel state using `Decimal` amounts so asset precision stays exact.
